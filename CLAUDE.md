# archipelagos — CLAUDE.md

## What this is

**archipelagos** (`archipelagosjournal.org`) is a peer-reviewed, born-digital academic journal devoted to Caribbean digital humanities. It publishes scholarly essays, digital project exchanges, and project reviews. The journal is trilingual: English, Spanish, and French. ISSN: 2689-842X. Published by Columbia University Libraries.

This repository (`archie-next`) is a forward migration of the original Jekyll 4.x site to **Eleventy 3.x**. It is a test environment using only issues 1–3. Once validated, the remaining issues will be integrated and the domain cut over from the Jekyll site.

## Architecture

- **Static site generator:** Eleventy 3.x (Node.js), output to `_site/`
- **Templates:** Nunjucks (layouts in `src/_layouts/`, includes in `src/_includes/`)
- **Markdown:** markdown-it with footnotes, attrs, and smart quotes (no Nunjucks pre-processing of markdown — `markdownTemplateEngine: false`)
- **Styles:** Plain CSS in `src/public/css/` — unchanged from the original Jekyll site
- **PDF pipeline:** Pandoc 3.x + lualatex, scripts in `utility/latex/`
- **i18n:** Custom — YAML files in `src/_i18n/{en,es,fr}.yml`, loaded as global data via `src/_data/i18n.js`, accessed in templates as `i18n[lang].global.key`

## Directory structure

```
src/
  _data/          # Global data: site.js, issues.js, i18n.js, pages.js
  _i18n/          # Translation YAML: en.yml, es.yml, fr.yml
  _includes/      # Nunjucks partials (head, sidebar, toc, footer, etc.)
  _layouts/       # Nunjucks layouts (default, article, project, page, info)
  public/         # CSS, fonts, favicon — served as-is
  assets/         # Pre-generated PDFs — served as-is
  images/         # Article images — served as-is
  issue01/        # Issue 1 markdown articles + directory data file
  issue02/        # Issue 2 markdown articles + directory data file
  issue03/        # Issue 3 markdown articles, parham/ and parham-process/ (passthrough)
  es/             # Spanish variants: index, info pages, issue-articles.11ty.js, issue-pages.njk
  fr/             # French variants: same structure as es/
  index.html      # Homepage (current issue only)
  issue-pages.njk # Generates /issue01.html, /issue02.html, /issue03.html via pagination
utility/
  latex/          # PDF pipeline: template.tex, journal.lua, convert.sh, makeIssues.sh
```

## Key conventions

**URL structure is frozen.** Article URLs must match the original Jekyll output exactly — DOI resolvers and citations depend on them. Permalinks use `.html` extensions (not directory indexes), e.g. `/issue01/haynes-mapping.html`.

**`date:` is reserved by Eleventy.** Human-readable publication dates live in `pubDate:` in front matter, not `date:`.

**Language variants** are generated by `src/es/issue-articles.11ty.js` and `src/fr/issue-articles.11ty.js` — JS class templates that paginate over `collections.allIssueArticles` and emit pages at `/es/issueXX/slug.html` and `/fr/issueXX/slug.html`.

**Interactives** (pure-HTML pieces like the Parham essay) are listed per-issue in `src/_data/issues.js` under an `interactives` array (title, author, url) and rendered in `toc.njk` after Essays. Their HTML files pass through Eleventy untouched via `addPassthroughCopy` + `eleventyConfig.ignores`.

**Current issue** is set in `src/_data/site.js` (`current: "issue03"`). The homepage shows only the current issue; past issues are accessible via individual pages (`/issue01.html` etc.) linked from a "Issues / Números / Numéros" nav at the bottom of the homepage.

## PDF pipeline

To generate PDFs for a single article:
```bash
bash utility/latex/makeSinglePaper.sh src/issue01/haynes-mapping.md
```

To regenerate all issues:
```bash
bash utility/latex/makeIssues.sh
```

Requires Pandoc ≥ 3.0 and lualatex (TeX Live). The template uses TeX Gyre Heros (open-source Helvetica Neue equivalent). To use licensed Helvetica Neue fonts, install `.ttf` files to `.fonts/` and uncomment the block in `utility/latex/template.tex`.

## Development

```bash
npm run serve   # local dev server at http://localhost:8888
npm run build   # production build to _site/
```

## Deployment

GitHub Pages via GitHub Actions (`build.yml`). Push to `main` builds and deploys automatically. PDF generation is a separate manual workflow (`pdf.yml`). The live domain `archipelagosjournal.org` still points to the old Jekyll site until validation of this repo is complete.
